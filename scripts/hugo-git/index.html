HUGO PUBLIC & CONTENT GIT

Hvad er poiten?

Poiten er altså at vi bruger GITHUB pages for at folk skal kunne se vores hjemmeside (uden vi betaler), hurtigt, nemt, og fra overalt.

Hvordan fungere det?

Vi vil gerne have vores HUGO site op på github, mest for os selv sådan vi kan få en fuld kopi af vores kildekode hvad end der sker - MEN, vi kunne også tænke os en live version af siden, en statisk version af samme hjemmeside. Til det kan vi bruge GITHUB pages, gratis hosting af html filer fra Github til et domaine. Siden Hele vores HUGO server / side er en generator og en server i sig selv, kan vi ikke blot uploade den til GITHUB pages (eftersom den kun fatter html og css (og ikke kan køre en server)).

Hvordan gør man?

Man har sådan set sat to repositories op, en til vores server, og en til vores statiske side. Hugo har en funktion der netop gør det nemt for os, så alt vi skal gøre for at få en offline og ren html - css side, er at skrive hugo i terminalen mens vi er i mappen. Så generere den en mappe kaldet PUBLIC, i vores server, hvor alt html er i. Det er mappen PUBLIC vi skal sync til github. Vores public repository skal man sætte op som en github page, altså navnet på vores repo er brugernavn.github.io, og aktiveret gennem github’s side.

Når det er gjordt, burde man kunne se en version af sin hjemmeside på sin github page aka. brugernavn.github.io, samt man skal kunne se hele sine server filer i et andet repository fra Github.

I sidste ende ser vores Hugo server (nogenlunde) sådan her ud. –> Root Mappe –> themes –> posts –> content –> data –> resources –> PUBLIC –> static
